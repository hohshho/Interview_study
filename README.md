## 1. 객체 지향 프로그래밍

###

    1. 정의
     - 인간 중심적 프로그래밍 패러다임
     - 프로그램 설계 방법론이자 개념의 일종
     - 현실 세계 사물 -> 객체
     - 어플리케이션 특징(객체의 상호작용) -> 프로그래밍(=추상화)

    2. 특징
     -

    3. 관련 용어
     -
    4. 장점
     - 코드 재사용 용이
     - 유지 보수 쉬움
     - 강한 응집력과 약한 결합력을 가진다
     - (응집력 : 기능을 수행하기 위해 얼마만큼의 연관된 책임과 아이디어가 뭉쳐있는지를 나타내는 정도) / 혼자 너무 많은 일 x
     - (결합력 : 프로그램의 요소가 다른 것과 얼마나 연결되어 있는지, 의존적인지)
    5. 단점
     - 설계시 많은 시간 필요
     - 객체들간의 통신하는 오버헤드로 처리속도가 상대적으로 느림

### 1) 질문 : 객체 지향 프로그래밍이 뭔가요?

- 답변 : 프로그램 설계 방법론이자 개념의 일종으로 인간 중심적 프로그래밍 패러다임 입니다. 현실 세계 사물을 추상화하여 객체를 만들고, 객체간의 상호작용을 통해 시스템을 구성하는 프로그래밍 방법입니다.

### 2) 질문 : 객체 지향 프로그래밍이 장, 단점이 뭔가요?

- 답변 : 장점으로는 코드 재사용 용이, 유지 보수가 쉽고 강한 응집력과 약한 결합력을 가집니다. 단점으로는 설계시 많은 시간이 필요하고, 객체간 통신하는데 오버헤드가 발생해 처리속도가 상대적을 느립니다.

## 2. RESTful API

###

    1. 정의
     - 소프트웨어 아키텍처의 한 형식으로 자원을 정의하고 자원에 대한 주소를 지정하는
       방법

    2. 특징
     - 지정한 주소를 HTTP Method를 통해 자원을 처리한다.
     - REST의 6가지 기본 원칙을 성실히 지킨 서비스 디자인을 RESTful하다라고 표현


    3. 관련 용어


    4. 장점
     - 기존 웹 인프라를 그대로 사용할 수 있다.
     - 쉽고, 빠르다
     - 세부적인 표현 문구를 JSON, XML등 다양한 언어를 이용하여 작성할 수 있다.

    5. 단점
     - 메소드 형태가 제한적이다.
     - 표준 규약이 없다.
     - HTTP 통신 모델에 대해서만 지원합니다.

    Tip!!
     - RESTful 하게 API를 디자인 한다는 것은 무엇을 의미하는가?
       ->

### 1) 질문 : RESTful API가 뭔가요?

- 답변 : 소프트웨어 아키텍처의 한 형식으로, 자원을 정의하고 자원에 대한 주소를 지정하는 방법입니다. HTTP Method를 통해 자원을 처리합니다.

### 2) 질문 : RESTful API의 장, 단점이 뭔가요?

- 답변 : 장점으로는 기존 웹 인프라를 그대로 사용할 수 있고, 쉽고, 빠릅니다. 또한 세부적인 표현 문구를 JSON, XML등 다양한 언어를 사용하여 작성이 가능합니다. 단점으로는 메소드 형태가 제한적이고, 표준 규약이 없습니다. 또한 HTTP 통신 모델에 대해서만 지원합니다.

## 3. TDD

###

    1. 정의
     - 테스트 주도 개발
     - (개발 테스트 단위가 함수단위로 매우 작아서 작성하는 거의 모든 함수가 테스트에 포함된다.)

    2. 특징
     -

    3. 관련 용어
     1) BDD : 시나리오 기반으로 테스트 케이스 작성
      - Given -> When -> Then
      - 어떤 상태 (Given) -> 어떤 상태 변화를 가했을 때(When) -> 기대하는 상태로 완료되어야 한다.(Then)

    4. 장점
     - 피드백이 많아진다.
     - (테스트를 통과하는 것으로 개발이 잘 진행되는지 자주 확인 가능해진다.)
     - 코드 복잡도가 낮아진다.

    5. 단점
     - 개발시간이 길어진다.
